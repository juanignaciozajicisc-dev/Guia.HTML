Fundamentos

HTML no es lenguaje de programación: es un lenguaje de marcado. Su tarea es comunicar al navegador qué significa cada contenido. <p> dice "esto es un párrafo", <h1> dice "esto es lo más importante", <button> dice "esto es accionable".

🏗️ Anatomía de una etiqueta

Casi todas las etiquetas abren, contienen algo, y cierran. Ejemplos: <p>Texto</p>, <div>Contenido</div>. Algunas, como <img> o <input>, nunca cierran porque no envuelven contenido.

⚙️ Atributos

Los atributos dan información o modifican comportamiento. href dice a dónde va el enlace. alt describe una imagen. class aplica estilos. aria-label dice qué es para lectores de pantalla.

<a href="inicio" class="boton">Link</a>

🎯 Semántica

Una etiqueta semántica comunica significado. <button> no es solo visual: el navegador sabe que se activa con teclado y Enter. <nav> explica que eso es navegación. <footer> señala el pie de página.

<button type="submit">Guardar</button>

Cómo editar HTML

Editar una página significa trabajar en tres capas independientes: la estructura con HTML, la apariencia con CSS y el comportamiento con JavaScript. Puedes cambiar una capa sin afectar las otras.

1️⃣ Cambia contenido

Edita el texto dentro de las etiquetas: títulos, párrafos, listas, botones y enlaces. El navegador los muestra tal cual lo dejas.

2️⃣ Cambia atributos

Modifica href para cambiar destinos, src para otras imágenes, alt para descripciones, class para aplicar estilos CSS diferentes.

3️⃣ Cambia estilos

Usa CSS en <style> para colores, tamaños, espacios, grillas, bordes, efectos hover y responsividad sin tocar el HTML.

4️⃣ Valida la estructura

Revisa cierres correctos, jerarquía de encabezados (h1 antes de h2), formularios con labels vinculados, y que cada página tenga un único <main>.

💡 Consejo clave: Si dudas entre <div> y una etiqueta semántica, pregunta qué significa ese bloque. ¿Es navegación? Usa <nav>. ¿Es independiente? Usa <article>. ¿Es un botón de verdad? Usa <button>, no un div estilizado.

Buenas prácticas

HTML fuerte es claro, accesible y fácil de mantener. El navegador perdona muchos errores, pero las personas, los buscadores y el código futuro agradecen precisión y significado.

✍️ Etiquetas por significado

No elijas una etiqueta solo por cómo se ve visualmente. Para apariencia usa CSS. Si una etiqueta tiene nombre específico (<nav>, <button>, <time>), úsala correctamente.

🖼️ Describe siempre los medios

Toda imagen informativa necesita alt descriptivo. Videos y audios importantes necesitan controles y subtítulos en idioma natural para que todos accedan.

📋 Formularios accesibles

Relaciona cada <label> con su campo usando for e id. Usa tipos HTML5 correctos como email, number, date para mejor experiencia.

⚠️ Evita etiquetas obsoletas

<font>, <center> y <marquee> pertenecen a otra época. CSS hace ese trabajo mejor y sin perder mantenibilidad.
